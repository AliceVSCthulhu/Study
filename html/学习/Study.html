// 异常捕捉
// 程序在运行过程中，存在一下无法预测的异常情况
try {
  // ToDo 正常运行的过程
} catch (err) {
  // 只要存在报错，try 代码块会立即结束运行，然后运行 catch 代码块
} finally {
  // 不管程序如何运行都会执行
}

function demo () {
  // 直接跳出函数执行
  // return 13;
  try {
      // try 内部存在 return，return之后会执行 finally
      return 13;
  } catch (err) {
      return err;
  } finally {
      // 函数体内，即使 return 跳出函数执行，也会执行此部分代码
      console.log(13213)
  }
}

过程化代码结构 

1. 所有代码都是从上往下执行的一个过程
2. 除循环外都执行一遍
3. 所有变量在当前作用域、代码块内或子代码块内生效。(默认的全局变量除外)
